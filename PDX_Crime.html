PDX Crime Map Dashboard¶

Portland Police Bureau publishes interactive Tableau dashboards with their crime data each month. I used the accompaninying raw data to recreate one. The most helpful feature for a resident IMO is the neighborhood heat map. In order to acheive this recreation, I had to find a geospacial file containing the polygon boundatries of each Portland neighborhood and merge it with the crime data before building the report in Tableau.

The geospacial file was in KML format and required the use of Fiona and Geopandas to read it into a workable format to merge with the PDX crime csv.

I'll outline the steps taken and also provide my Tableau dashboard at the bottom. Click here to skip the code and launch Tableau.

There are 25 more neighborhood names or variations on geo_data than the crime_data list. Not only are some of them intersections of two different neighborhoods, several have different names all together. I can't trust they can be combined correctly this way.

Consensus¶

I'll run the spatial join and manually insert the Polygon data into the 7 rows with a Neighborhood name but no lat/lon coordinates. The other 1,140 are bad data for the purpose of linking them to a geo location. They should stick around because we can still use them towards the total call volume for each month they were recorded in.
